## 1. หลักการออกแบบ (Design Principles)
ยึดหลักจาก SRS ให้ UX สะท้อน 4 ค่านิยมของโครงการ:
1. **Evidence-first** — คะแนนต้องอยู่ติดกับหลักฐานเสมอ (ไม่มีคะแนนลอย) → ใช้ layout **side-by-side** (แผน ↔ เกณฑ์)
2. **Human-in-the-loop ชัดเจน** — แยกสายตา "AI เสนอ" (draft, สีเทา/ป้าย AI) ออกจาก "CAM ยืนยัน" (สีเข้ม/actionable)
3. **กัลยาณมิตร ไม่ตัดสิน** — โทนสีเชิงบวก, ภาษาสุภาพ, เน้น Growth ไม่เน้นตำหนิ
4. **ลดภาระ CAM** — ตรวจ 1 แผนจบใน ≤10 นาที → เดินตามลำดับ C1→C5 แบบ guided

## 2. Design System — ยึดตาม DigiNest LMS (KSU)
> อ่านจากสไตล์จริงของ `craftai.ksu.ac.th/lms` (Next.js + Tailwind, ตัวแปรสี HSL/OKLCH, รองรับ light/dark) เพื่อให้ ALPR กลมกลืนเป็นโมดูลในระบบเดียวกัน

### 2.1 Design Tokens (ดึงจาก LMS จริง)
| Token | Light | Dark | หมายเหตุ |
|-------|-------|------|----------|
| `--hue` | 225 | 225 | ฐานสีน้ำเงิน-อินดิโก้ทั้งระบบ |
| Background | `hsl(46 42% 96%)` ครีมอุ่น ≈ `#F8F5EC` | `hsl(225 25% 12%)` ≈ `#171B26` | พื้นหลังหน้า |
| Card / Surface | `#FFFFFF` | `hsl(225 22% 16%)` | การ์ดขาวบนพื้นครีม |
| **Primary** | `hsl(225 40% 45%)` ≈ `#455CA1` | `hsl(225 40% 65%)` ≈ `#8496CE` | ปุ่ม/ยืนยัน (foreground = ขาว) |
| Foreground (text) | `hsl(225 20% 22%)` ≈ `#2D3243` | `hsl(225 10% 93%)` | ข้อความหลัก |
| Muted / MutedFg | `hsl(225 20% 94%)` / `hsl(225 15% 50%)` | — | พื้นรอง/ข้อความรอง |
| Border | `hsl(225 20% 88%)` ≈ `#DADEE8` | `hsl(225 18% 26%)` | เส้นขอบอ่อน |
| Ring (focus) | `hsl(225 40% 65%)` | — | โฟกัส a11y |
| Success | `oklch(0.58 0.14 155)` เขียว | — | ระดับ 4 / สำเร็จ |
| Destructive/Danger | `hsl(15 80% 50%)` ส้ม-แดง | — | ระดับ 1 / เตือน |

### 2.2 Typography (ตรงกับ LMS)
- **หัวเรื่อง (Display/Heading):** `Prompt` — ฟอนต์ไทยทรงเรขาคณิต
- **เนื้อหา/UI:** `IBM Plex Sans Thai` (fallback: IBM Plex Sans → system-ui)
- ต้องโหลด font ผ่าน `<basePath>` (ดูข้อ 7)

### 2.3 Shape & Elevation (ตรงกับ LMS)
- **Radius:** `--r-sm 8px` · `--r-md 16px` · `--r-lg 20px` · `--r-xl 24px` (ค่า default `--radius: 1rem`) → มุมโค้งเยอะ ดูนุ่มนวล
- **Shadow:** เงา 2 ชั้นแบบนุ่ม (sm/md/lg) โทน oklch โปร่ง — การ์ดลอยเบา ๆ ไม่แข็ง

### 2.4 การ map สีสถานะประเมิน (ให้เข้าธีม 225)
เพื่อคงความหมาย 4 ระดับแต่ไม่หลุดจากพาเลตต์ LMS:
| ระดับ | สี | อิงจาก |
|-------|-----|--------|
| 4 ต้นแบบ | เขียว `--success` | token LMS |
| 3 เชี่ยวชาญ | น้ำเงิน `--primary` (hue 225) | token LMS |
| 2 บ่มเพาะ | เหลือง-อำพัน (amber) | เสริม (คงความหมาย warning-soft) |
| 1 เริ่มจุดประกาย | ส้ม-แดง `--destructive` | token LMS |
| AI-draft | เทา (muted) + ป้าย ✨ "AI เสนอ" | muted token |
| ⚠️ ห้ามใช้สอน (9–12) | แถบ `--destructive` เด่น | token LMS |

### 2.5 อื่น ๆ
- **Dark mode:** รองรับ (ใช้ `.dark` / `[data-theme=dark]` เหมือน LMS)
- **Responsive:** Desktop (หลัก) + Tablet แนวนอน (CAM ตรวจในโรงเรียน)
- **A11y:** contrast ≥ 4.5:1 · ปุ่ม ≥ 44px · focus ring (`--ring`) · keyboard navigable · label ไทยครบ

### 2.6 ทิศทางสไตล์ที่ยืนยันแล้ว (จากหน้า LMS จริง)
> **อัปเดต v1.3:** ยึดโทน **Light & Airy** ตามหน้า `craftai.ksu.ac.th/lms` ที่เห็นจริง (ยกเลิกแนว Dark Glassmorphism)
- **สถาปัตยกรรม (Architecture):** **Multi-page Application (MPA)** แยกไฟล์ HTML แต่ละหน้า (Separate Page Design)
- **ธีมหลัก:** **Light mode** — สว่าง โปร่ง สะอาด
- **พื้นหลัง:** ไล่เฉด **ครีมอุ่น → ฟ้าอ่อน** (`#F3EFE6` → `#EAF1F7`) นุ่มนวล ไม่มีการเคลื่อนไหว
- **แถบหัว:** โลโก้พันธมิตรบนแถบครีม + คำอธิบายโครงการ (เหมือน LMS)
- **การ์ด:** พื้นขาว มุมโค้ง 16–24px เงานุ่ม (soft shadow) — ไม่ใช้ glass/blur
- **Layout หน้า Login:** **สองคอลัมน์** — ซ้าย Hero (badge + หัวเรื่องใหญ่ Prompt + การ์ดฟีเจอร์ 3 ใบ) · ขวา การ์ดล็อกอิน
- **โค้ดจริง:** ดูโฟลเดอร์ `design/` (§9)

## 3. Navigation & Roles (Multi-page Architecture)
โครงสร้างไฟล์ถูกแยกส่วนตามหน้า (Separate Page Design) ดังนี้:
```text
เข้าสู่ระบบ (`index.html`) ──► เลือก/ตรวจบทบาท
   ├── CAT ──► `cat_upload.html` (หน้า 2) ──► `cat_results.html` (หน้า 3)
   └── CAM ──► `cam_queue.html` (หน้า 4) ──► `cam_evaluate.html` (หน้า 5) ──► `cam_summary.html` (หน้า 6)
```
สถานะแผน (แสดงเป็น badge ทุกที่): `อัปโหลดแล้ว → AI กำลังวิเคราะห์ → รอ CAM ตรวจ → ประเมินเสร็จ`

### หน้า 1 — เข้าสู่ระบบ (Login) — โครงสร้างตาม DigiNest LMS
**เป้าหมาย:** ยืนยันตัวตน + เข้าสู่ workspace ตามบทบาท
**อ้างสไตล์จริง:** LMS ใช้ layout กึ่งกลางคอลัมน์เดียว = แถวโลโก้พันธมิตร → ชื่อ/คำอธิบายโครงการ → การ์ดฟีเจอร์ 3 ใบ → ฟอร์มล็อกอิน (คงรูปแบบเดียวกันเพื่อความต่อเนื่อง)
```
┌──────────────────────────────────────────────────────┐
│  [MHESI] [อปท.] [Saturday] [Craft AI] [TDRI] [KSU]    │ ← แถวโลโก้ (เหมือน LMS)
│                                                        │
│              DigiNest LMS · โมดูล AIPACK               │ ← Prompt font
│      ระบบตรวจประเมินแผนการจัดการเรียนรู้ AIPACK          │
│         พัฒนา RL & CT ด้วย AI · จ.กาฬสินธุ์             │
│                                                        │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐         │ ← 3 การ์ดฟีเจอร์
│  │ 📄 อัปโหลด  │ │ ✨ AI ช่วย   │ │ 👥 หลายบทบาท│         │   (มุมโค้ง 16px,
│  │ แผนการสอน   │ │ ประเมิน      │ │ CAT · CAM   │         │    เงานุ่ม, การ์ดขาว)
│  └────────────┘ └────────────┘ └────────────┘         │
│  ┌──────────────────────────────────────────┐         │
│  │  เข้าสู่ระบบ                                │         │
│  │  ┌────────────────────────────────────┐   │         │
│  │  │  [G]  เข้าสู่ระบบด้วย Google         │   │ ← ปุ่มเดียว
│  │  └────────────────────────────────────┘   │         │
│  │  ใช้บัญชี Gmail/Google Workspace           │         │
│  │  ของสถานศึกษา (โดเมนที่อนุญาต)             │         │
│  └──────────────────────────────────────────┘         │
│   บทบาทกำหนดอัตโนมัติจากบัญชี (CAT/CAM)                 │
└──────────────────────────────────────────────────────┘
```
- **การล็อกอิน:** **Sign in with Google (OAuth) เท่านั้น** — ไม่มีช่องอีเมล/รหัสผ่าน (ตาม SRS FR-1.1). กดปุ่ม → หน้ายินยอมของ Google → callback กลับเข้าระบบ
- **ปุ่ม Google:** ทำตาม Google Branding Guidelines (โลโก้ G สี่สีบนพื้นขาว/ปุ่มเข้ม, ข้อความ "เข้าสู่ระบบด้วย Google") — บนธีม glassmorphism วางเป็นปุ่มทึบอ่านง่าย ตัดกับการ์ดกึ่งโปร่งใส
- **สไตล์:** พื้นไล่เฉดครีม→ฟ้าอ่อน (ตาม §2.6) · การ์ดขาวมุมโค้ง 16–24px เงานุ่ม · ปุ่ม primary อินดิโก้ `#455CA1` · หัวเรื่องฟอนต์ Prompt · เนื้อหา IBM Plex Sans Thai
- **States:**
  - ปกติ · กำลังเปลี่ยนเส้นทางไป Google (spinner บนปุ่ม)
  - **ปฏิเสธโดเมน** — อีเมลนอก allow-list: แสดง banner สี `--destructive` "บัญชีนี้ไม่ได้รับอนุญาต โปรดใช้อีเมลของสถานศึกษา" + ปุ่มลองใหม่
  - **ผู้ใช้ใหม่รอกำหนดบทบาท** (`pending_role`): เข้าได้แต่แสดงหน้ารอ admin/CAM กำหนดบทบาท
  - session หมดอายุ → กลับหน้านี้พร้อมข้อความแจ้ง
- **หมายเหตุ:** Google ใช้ยืนยันตัวตนเท่านั้น — **บทบาท (CAT/CAM) ระบบเป็นผู้กำหนด** ไม่ได้มาจาก Google (กัน CAT ปลอมเป็น CAM)

## 5. User Flows (End-to-end)
```
CAT: เข้าระบบ → อัปโหลดแผน+กรอกข้อมูล → ส่งเข้าประเมิน
      → (ระบบ: สกัดข้อความ/OCR → เช็กลิสต์ → AI ให้คะแนน+Evidence)
      → รอผล

CAM: เข้าระบบ → เห็นคิว → เลือกแผน → [หน้า5] ยืนยัน/แก้ C1–C5 ทีละข้อ
      → [หน้า6] เขียน Strengths/Growth → ลงนาม → ปิดผล

CAT: ได้แจ้งเตือน → [หน้า3] ดูคะแนน+Feedback → ดาวน์โหลด PDF
      → (ถ้าคะแนนต่ำ) อัปโหลดฉบับปรับปรุง v2 → วนรอบใหม่
```

## 6. สถานะระบบร่วม (Global States) ต้องออกแบบทุกหน้า
| สถานะ | การแสดงผล |
|-------|-----------|
| Loading | skeleton/ spinner + ข้อความไทย |
| Empty | ภาพประกอบ + คำชวนกระทำ (เช่น "ยังไม่มีแผน เริ่มอัปโหลด") |
| Error | ข้อความชัดเจน + ทางแก้ + ปุ่มลองใหม่ |
| AI ไม่พบหลักฐาน | ป้าย "ไม่พบหลักฐาน" + ค่าเริ่มระดับต่ำ (ห้ามเดาสูง) |
| Session หมดอายุ | modal ให้เข้าสู่ระบบใหม่ (cookie scope = `<basePath>`) |
| Offline/timeout AI | แจ้ง "ประเมินไม่สำเร็จ ลองใหม่" ไม่บันทึกคะแนนมั่ว |

## 7. ข้อพิจารณาเรื่อง Base Path (จาก SRS NFR-9)
> **ยืนยันจากระบบจริง:** DigiNest LMS เป็น Next.js ที่รันใต้ basePath `/lms` (asset ที่ `/lms/_next/...`, โลโก้ที่ `/lms/logos/...`) — ALPR ต้องเดินตามรูปแบบเดียวกัน เช่น `/aipack` หรือเป็น sub-route ใต้ `/lms`
- ทุกลิงก์ในหน้า (nav, ปุ่มดาวน์โหลด PDF, asset **ฟอนต์ Prompt/IBM Plex Sans Thai**, โลโก้) ต้องขึ้นต้นด้วย `<basePath>`
- ปุ่ม "ดาวน์โหลดรายงาน" และลิงก์ในอีเมลแจ้งเตือน CAT ต้องเป็น absolute URL ที่ประกอบจากโดเมน + `basePath`
- Cookie session ตั้ง `Path=<basePath>` เพื่อไม่ชนแอปอื่นบนโดเมนเดียวกัน (สำคัญมากเพราะอยู่โดเมนเดียวกับ LMS `/lms`)
- **Google OAuth callback ต้องรวม `basePath`:** redirect URI ที่ลงทะเบียนใน Google Cloud Console = `https://<domain>/<basePath>/api/auth/callback/google` (ถ้า basePath ผิด callback จะ 404 — เป็นจุดพลาดที่พบบ่อยเมื่อ deploy ใต้ sub-path)
- **แนะนำ stack:** Next.js (App Router) + Tailwind + shadcn/ui ให้ตรงกับ LMS เพื่อ reuse token/ธีมและรองรับ dark mode ทันที
